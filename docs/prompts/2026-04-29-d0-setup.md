## Decisión y tradeoffs

| Decisión                                          | Alternativas                                  | Tradeoff aceptado                                                                                                                            |
| ------------------------------------------------- | --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **fetch nativo** sobre axios                      | axios (más popular, interceptors first-class) | Cero deps, contrato Web Platform, control total del error model. Cuesta ~50 líneas de wrapper.                                               |
| **Tailwind v4** zero-config (`@tailwindcss/vite`) | Tailwind v3 con `tailwind.config.js`          | v4 es más rápido y elimina archivo de config. Riesgo: ecosistema (shadcn) aún migrando — mitigado eligiendo Headless UI en su lugar.         |
| **Headless UI + Lucide + Sonner**                 | shadcn/ui CLI · MUI · Chakra                  | shadcn CLI hoy asume TS+Tailwind v3. Headless UI es del mismo equipo que Tailwind, accesible, ligero. MUI/Chakra rompen el target de bundle. |
| **`@theme {}` en CSS** para tokens                | `tailwind.config.js` con `theme.extend`       | Tailwind v4 way; tokens como CSS variables nativas; integra con Figma tokens.                                                                |
| **`clsx + tailwind-merge` (`cn()`)**              | classnames                                    | tailwind-merge resuelve conflictos de utilities (e.g. `p-2 p-4` → `p-4`). Estándar de facto.                                                 |
| **JSDoc en httpClient.js**                        | TS interno                                    | Mantiene "JS puro" (regla del equipo) con tipado documentado.                                                                                |
| **CLAUDE.md como guía del agente**                | Solo confiar en CLAUDE.md global del usuario  | El proyecto tiene reglas específicas (no axios, no TS, deadline) que no están en el global.                                                  |
| **`esbuild.jsx: 'automatic'`** en vite.config.js  | Importar React explícito en cada test         | El plugin React 6 + Vitest 2 no aplicaba el transform automático en algunos casos; setear esbuild lo arregla globalmente.                    |

- MUI / Ant Design / Chakra (tamaño bundle, dirección de tokens).
- Bootstrap (fuera de la dirección visual).
